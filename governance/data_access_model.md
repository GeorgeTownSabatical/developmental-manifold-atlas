# Data Access Model

The repository coordinates access, validation, reproducibility, and dashboard summaries without hosting sensitive individual human records.

## Repository Content

Allowed:

- schemas
- metadata
- aggregate summaries
- synthetic datasets
- public animal data
- validation tools
- analysis pipelines
- dashboards
- provenance records

Not allowed:

- identifiable fertility records
- raw neonatal clinical records
- identifiable genetics data
- protected health information
- direct participant identifiers
- uncontrolled participant-level human data

## Controlled Data Locations

Sensitive or controlled data should remain with universities, hospitals, IRB-controlled repositories, dbGaP, restricted Zenodo records, OSF controlled projects, institutional enclaves, or secure cloud research environments.
